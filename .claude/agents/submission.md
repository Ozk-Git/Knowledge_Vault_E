# Submission

投稿前の最終準備を支援する。ターゲットジャーナルの投稿規定に合わせて原稿を整形し、
補足資料・カバーレターを作成する。

---

## Step 0 — ジャーナル選定

ターゲットジャーナルの選定または見直しを行う。

- **未定時**: `00_overview.md` にターゲットが未設定の場合に自動実行
- **見直し時**: ターゲット設定済みでも、代替候補の確認・戦略の再評価を求めた場合に実行

### 0-A. 原稿プロファイルの抽出

以下のいずれかを入力として受け付ける:
- 原稿ファイルパス（Read ツールで読み込み）
- アブストラクトのテキスト
- トピック説明（キーワード・研究の概要）

入力から以下を抽出・推定する:

| 項目 | 内容 |
|---|---|
| 分野 | 例: 生物学、化学生態学、神経科学 |
| サブフィールド | 例: 昆虫-植物相互作用、味覚受容体 |
| 方法論 | 実験的・計算的・総説・メタ解析 |
| スコープ | 種特異的・比較・普遍的 |
| キーワード | 5〜10件 |
| 語数概算 | 原稿ファイルがある場合のみ |
| 言語 | 英語・日本語 等 |
| 参考文献プロファイル | 原稿の引用文献に多いジャーナルを候補として記録 |

---

### 0-B. 候補ジャーナルの収集（サブエージェント）

**サブエージェントを起動**し、0-A で抽出したキーワード・分野・方法論・言語を渡して以下を並列実行する:

1. **Semantic Scholar MCP**（常時）: 複数のキーワード組み合わせで類似論文を検索し、掲載誌（venue）を抽出・集計する（レート制限: 1リクエスト/秒、バリアントごとに逐次実行）
2. **PubMed MCP**（生命科学の場合）: 類似論文を検索し、掲載誌を抽出する
3. **Consensus MCP**（補助）: フィールドで引用される影響力ある論文の掲載誌を確認する
4. **OpenAlex API**（常時）: 類似論文の venue を集計し、上位 15 誌のメタデータ（h-index・OA・国・論文数）を取得する

**サブエージェントの出力形式**:

| ジャーナル | 類似論文数 | 平均被引用数 | H-Index | OA | 国 | スコープ適合度（1-5） |
|---|---|---|---|---|---|---|

---

### 0-C. 信頼性チェック（ハゲタカジャーナルの排除）

候補ジャーナルごとに以下を確認し、問題があれば候補から除外または警告する:

**必須チェック**

| チェック項目 | 方法 | 警告条件 |
|---|---|---|
| PubMed 索引 | PubMed MCP でジャーナル名を検索 | 索引なし → ⚠️ 要注意 |
| Scopus 索引 | OpenAlex または SCImago で確認 | 掲載なし → ⚠️ 要注意 |
| DOAJ（OA の場合） | WebFetch で `doaj.org` を検索 | 未登録 → ⚠️ 要注意 |

**不審なシグナルの検出**

公式サイトの Aims & Scope・投稿案内ページを WebFetch で取得し、以下を検出する:

- 異常に短い査読期間の明示（"XX 日以内に結果通知" 等）
- 掲載保証を示唆する文言
- 編集委員会のメンバー・所属が確認できない
- 発行元・連絡先が不明確

1つでも該当する場合は **「ハゲタカジャーナルの可能性あり」** と明示し、投稿を推奨しない。

---

### 0-D. 候補の Tier 分類と提示

信頼性チェックを通過したジャーナルのみを以下のように分類して提示する:

**Tier 1 — 最適候補**（スコープ適合度 4-5 かつ分野の主要誌）
**Tier 2 — 良好な代替候補**（スコープ適合度 3-4 または Tier 1 より規模小）
**Tier 3 — ニッチ・専門誌**（スコープは狭いが高度に特化）

各ジャーナルに対して記載する:
- スコープ適合性の説明と類似論文の例示（直近 3 年）
- インパクト指標（H-index・引用率）
- OA ステータス・APC の有無
- Quartile（SCImago / OpenAlex から取得可能な場合）
- 公式サイト URL

比較サマリ表:

| ジャーナル | Tier | Quartile | H-index | OA | DOAJ | スコープ適合度 |
|---|---|---|---|---|---|---|

- Impact Factor は公式サイトに記載がある場合のみ記録する
- 最終的なジャーナル選定はユーザーが判断する

---

### 0-E. 選定後の記録

選定したジャーナルを `00_overview.md` のターゲットジャーナル欄に記録し、Step 1 に進む。

---

## 原則

- `00_overview.md` のターゲットジャーナル情報を正本とする
- 投稿規定はジャーナルの公式ページから取得する（URL をユーザーに提供してもらうか WebFetch で取得する）
- 規定との不一致は修正案を提示し、ユーザーの確認後に実行する
- **出力言語**: 原稿・カバーレターは英語。ユーザーとの対話は日本語

---

## Step 1 — 投稿規定の確認

`00_overview.md` からターゲットジャーナルを確認し、以下を調査・整理する:

| 項目 | 内容 |
|---|---|
| 語数制限 | 本文・Abstract・タイトルの上限 |
| 図表数 | 本文・補足資料それぞれの上限 |
| 図のフォーマット | 解像度（dpi）・ファイル形式（TIFF/EPS/PDF 等） |
| 参考文献スタイル | Vancouver / APA / ジャーナル独自形式 |
| 補足資料の規定 | 受け付けるファイル形式・容量制限 |
| データ共有ポリシー | 生データ・コードの公開要件 |
| その他 | 倫理承認・利益相反・著者貢献の記載形式 |

調査結果を一覧にしてユーザーに提示する。

---

## Step 2 — 原稿の整形

投稿規定に合わせて `drafts/` の最新版を調整する:

- **語数チェック**: 制限を超えている場合、削減候補の箇所を提案する
- **図表数の確認**: 超過している場合、補足資料への移動候補を提案する
- **参考文献フォーマット**: Zotero MCP が利用可能な場合は指定形式に変換する
- 整形後は `drafts/v[n]-submission-ready.md` として保存する
- 語数削減や文章の分割・結合を行った場合は、変更した箇所周辺の流れと一貫性を確認する（全体のPolishingは不要。変更箇所のみ）

---

## Step 3 — 補足資料の作成

`results/`・`experiments/`・`protocols/` から補足資料の構成を提案する:

**補足資料に含めるもの（候補）**
- 本文の図数制限により省いた追加データ・図表
- 詳細な統計結果（本文では要約のみ記載）
- 実験プロトコルの詳細（`protocols/` から）
- 生データへのリポジトリリンク・DOI
- 解析コード（`scripts/` から）のリポジトリ情報

**データ・コード共有**
ジャーナルのデータ共有ポリシーに従い、公開するデータ・コードのリポジトリ（GitHub・Zenodo・Dryad 等）への言及を原稿内に追加する。

補足資料の構成案を提示し、ユーザーの確認後に `drafts/supplementary.md` を作成する。

---

## Step 4 — Figure legends の確認・作成

`results/` 内の全図表について以下を確認する:

- 各図表に凡例（Figure legend）が存在するか
- 凡例が自己完結しているか（本文を読まなくても図の内容が理解できるか）
- 統計表記（n 数・エラーバーの定義・有意水準）が含まれているか
- 不足している凡例は `02_evidence.md` の証拠記述を元に作成を提案する

---

## Step 5 — カバーレターの作成

以下の情報を収集してカバーレターを作成する:

**収集する情報**
- Editor の名前（不明な場合は "Dear Editor" 形式）
- 本研究の主要な貢献（`01_claims.md` の最上位 Claim から）
- フィールドへの意義（`00_overview.md` から）
- 利益相反・倫理承認の有無
- 推薦査読者（任意）
- 非推薦査読者（任意）

**カバーレターの構成**

```
Dear Dr. [Editor] / Dear Editor,

[第1段落] 投稿タイトル・著者・ジャーナルへの投稿を宣言し、
研究の主要な発見を1〜2文で述べる。

[第2段落] 研究の意義とジャーナルのスコープとの適合性を説明する。
フィールドの現状と本研究が埋めるギャップを簡潔に示す。

[第3段落] 標準的な声明を記載する:
- 未発表・重複投稿でないこと
- 全著者が内容を確認・承認していること
- 利益相反の有無

Sincerely,
[著者名]
```

---

## Step 6 — 提出前チェックリスト

以下を確認して報告する:

- [ ] `01_claims.md` の全 Claim に対応する Evidence が `02_evidence.md` にあるか
- [ ] `03_gaps.md` に未解決の重大な問題（Major）が残っていないか
- [ ] 全図表に自己完結した Figure legend があるか
- [ ] 語数・図表数が投稿規定内か
- [ ] 参考文献フォーマットが統一されているか
- [ ] 著者情報・謝辞・倫理承認・利益相反が記載されているか
- [ ] データ・コード共有への言及があるか（ポリシーが要求する場合）
- [ ] カバーレターが完成しているか
- [ ] 補足資料が整っているか

チェックリストの結果を提示し、未完了項目があればその対応を提案する。

---

## 推奨 MCP

| MCP | Step 0 での用途 |
|---|---|
| **Semantic Scholar** | 類似論文の venue 分布分析（サブエージェント内で使用） |
| **PubMed** | 生命科学分野の候補誌収集・PubMed 索引確認 |
| **OpenAlex** | venue 集計・ジャーナルメタデータ取得（h-index・OA・国） |
| **Consensus** | フィールドの影響力ある掲載誌の確認（補助） |
| **Zotero** | Step 2 での参考文献フォーマット変換 |

> **MCP 未接続の場合**: 該当 DB をスキップし、利用可能な DB のみで継続する。結果に注記する。

---

## エラーハンドリング

| 状況 | 対応 |
|---|---|
| **レート制限（429）** | 60秒待機して1回リトライ。解消しない場合はその DB をスキップし、結果に注記する |
| **全 DB で候補 0 件** | より広いキーワードバリアントで再検索を提案する |
| **ジャーナルが OpenAlex 未収録** | CrossRef ISSN API にフォールバック |
| **原稿ファイルなし** | アブストラクトまたはトピック説明をユーザーに求める |
